Câu lệnh if

Cú pháp:


if <biểu-thức-Boolean>:
    <statement>
    ...

Ví dụ:


x = 10
if x % 2 == 0:
    print('even')

if–else

Cú pháp:


if <biểu-thức-Boolean>:
    <statement>
    ...
else:
    <statement>
    ...

Ví dụ:


if x % 2 == 0:
    print('even')
else:
    print('odd')

Cấu trúc vs. Luồng

Cấu trúc chương trình

  • Cách mã được trình bày trong tệp.
  • Thứ tự dòng, mức thụt lề.
  • Định nghĩa các khả năng qua nhiều lần chạy.

Luồng chương trình

  • Thứ tự thực thi lệnh khi chạy.
  • Không trùng với cấu trúc; có thể bỏ qua nhánh.
  • Định nghĩa điều gì xảy ra trong một lần chạy.

Ví dụ: Cấu trúc vs. Luồng


# odd_even.py
x = 10
if x % 2 == 0:
    print('even')
else:
    print('odd')
        

> python odd_even.py
even

Chuỗi print('odd') tồn tại trong cấu trúc nhưng có thể không chạy trong luồng.:contentReference[oaicite:8]{index=8}

Lưu ý khi dùng elif

  • Số lượng elif không giới hạn; else là tùy chọn.
  • Điều kiện được kiểm tra theo thứ tự; gặp điều kiện đúng thì bỏ qua phần sau.
  • else có nghĩa là “tất cả điều kiện đều sai”.:contentReference[oaicite:12]{index=12}

Nhiều if liên tiếp vs. if–elif

Nhiều if


x = 0
if x == 0:
    print("x is 0!")
if x == 0:
    print("still 0!")
if x == 0:
    print("even still 0")

if–elif


x = 0
if x == 0:
    print("x is 0!")
elif x == 0:
    print("still 0!")
elif x == 0:
    print("even still 0")

Chuỗi if–elif đảm bảo chỉ một nhánh được chọn.:contentReference[oaicite:13]{index=13}

Điều kiện kết hợp


# mặc gì?
if raining and freezing:
    print('Wear a waterproof coat.')
elif raining and not freezing:
    print('Bring an umbrella.')
elif not raining and freezing:
    print('Wear a warm coat!')
else:
    print('A sweater will suffice.')

def what_to_wear(raining, freezing):
    if raining:
        if freezing:

        else:
            return 'Umbrella'
    else:
        if freezing:
            return 'Warm coat'
        else:
            return 'Sweater'

Tổng kết

  • if / if–else / if–elif–else điều khiển luồng thực thi.
  • Biến tạo trong nhánh tồn tại sau khối if nếu đã được gán.
  • Gỡ lỗi bằng trace (luồng) & watch (giá trị).
  • Kiểm thử bao phủ mọi nhánh, kể cả if lồng nhau.
  • Hàm: định nghĩa, đối số, trả về, dùng từ module.
